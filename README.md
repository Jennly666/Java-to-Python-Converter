## Классы, конструкторы, методы

* Объявления классов с телом и (игнорируемыми) модификаторами.
* Детекция конструкторов по имени класса (с модификаторами перед именем).
* Делегирование конструкторов:

  * `this(...)` → `self.__init__(...)`
  * `super(...)` → `super().__init__(...)`
* Присваивания в конструкторе:

  * голый идентификатор слева в `Assign` превращается в `self.<name> = ...`
* Статические методы:

  * `static` → `@staticmethod`, без `self`
* Сигнатуры:

  * типы параметров/возврата маппятся в Python-аннотации
  * массивы Java `T[]` → `list[T]`
  * `void` → `None`

## Поля и локальные объявления

* Поля класса (в т.ч. `static final`) → class-level атрибуты с аннотациями и инициализацией по умолчанию.
* Простые инициализаторы полей (литералы, вызовы, выражения).
* Локальные переменные, в т.ч. когда тип — `IDENTIFIER` (например, `String s = ...`).
* Инициализация массивов литералами `{ ... }` → Python-списки.

## Операторы и выражения

* Бинарные: `+ - * / %`, сравнения `>, <, >=, <=, ==, !=`, логические `&&/||` → `and/or`, битовые `& | ^ << >>`.
* Унарные: `!` → `not`, `~`, унарные `+/-`.
* Инкремент/декремент:

  * префикс/постфикс `++/--` → `+= 1` / `-= 1` (в нужном контексте).
* Тернарный `cond ? a : b` → `a if cond else b`.
* Члены/вызовы: `obj.field`, `obj.method(a, b)`, цепочки доступа.
* Спец-идентификаторы: `this` → `self`, `super` → `super`, `true/false/null` → `True/False/None`.

## Потоки управления

* `if / else if / else` → корректные `if / elif / else`.
* `switch` → `match/case` (без `break` внутри `case`).
* Циклы:

  * Классический `for(init; cond; upd)`:

    * распознаётся как `range(start, end, step)` когда возможно;
    * иначе fallback: `init; while(cond): ...; update`.
  * foreach `for (T x : expr)` → `for x in expr:`
  * `while` / `do-while` → прямые эквиваленты в Python.
* `break`, `continue`, `return` — в нужных местах.

## Ввод/вывод и утилиты

* `System.out.println(x)` → `print(x)`
* `System.out.print(x)` → `print(x, end='')`
* `List.of(a, b, ...)` → Python-список `[...]`

## Лексер

* Комментарии `//` и `/* ... */` и пробелы игнорируются.
* `LT/LA/consume` в `TokenStream` не вылетает за EOF.
* Парсер устойчив к лишним токенам и аккуратно скипает неожиданное.
